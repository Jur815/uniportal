# UniPortal Demo User Manual

## Institutional Student Information & Enrollment Management System

Prepared for demo/testing purposes.

This guide explains how university administrators, registrars, and students can use the current UniPortal institutional demo during a live presentation or internal testing session.

## 1. System Overview

UniPortal is a demo Student Information and Enrollment Management System designed to show how a university can manage core academic registration workflows.

The current demo supports:

- Student account access and protected login.
- Admin dashboard for institutional overview.
- Academic structure setup for faculties, departments, programs, and courses.
- Course management with active/inactive course status.
- Student management and academic profile review.
- Academic session and enrollment window control.
- Student course browsing and enrollment request submission.
- Registrar enrollment review and approval workflow.
- Enrollment decision audit log.
- Approved enrollment slip printing.
- Student academic records foundation with grades, GPA, credits, and remarks.
- Demo readiness page for presentation preparation.

UniPortal is currently intended for institutional demo and pilot discussion. It should be presented as a focused registration and academic workflow MVP, not yet as a complete production SIS.

## 2. User Roles

### Admin

The Admin role manages the institutional setup and overall system readiness.

Admins can:

- View the admin dashboard.
- Open the demo readiness page.
- Manage students.
- Manage academic sessions.
- Manage faculties and departments.
- Manage courses.
- View course enrollment counts.
- Access enrollment review pages.
- Access academic records.

### Registrar

The Registrar role manages enrollment approval and academic registration operations.

Registrars can:

- View the registrar dashboard.
- Review pending enrollments.
- Approve enrollments.
- Reject enrollments.
- Return enrollments for correction.
- View enrollment decision audit logs.
- View and update academic records where enabled.

### Student

The Student role uses UniPortal to manage their own registration and academic information.

Students can:

- Login to the student dashboard.
- Update allowed profile fields.
- Browse available courses.
- Submit course enrollment requests.
- View enrollment status.
- Print approved enrollment slips.
- View academic records.

## 3. Demo Login Credentials

Demo credentials are generated by the seed script and displayed in the terminal after running:

```bash
npm run seed:demo
```

Do not hardcode or publish real passwords in frontend code, documentation, screenshots, or public repositories.

Typical demo account emails include:

- Admin: `admin@uniportal.demo`
- Registrar: `registrar@uniportal.demo`
- Student: `john.student@uniportal.demo`
- Student: `amina.student@uniportal.demo`

Use the passwords printed by the seed command.

## 4. Admin Guide

### Login as Admin

1. Open the UniPortal frontend URL.
2. Go to the login page.
3. Enter the demo admin email.
4. Enter the password printed by the seed script.
5. Click login.

### View the Admin Dashboard

1. After login, open `/dashboard`.
2. Review the institutional snapshot.
3. Confirm the active academic session is visible.
4. Review KPI cards for students, courses, faculties, departments, and enrollments.
5. Review recent students, recent courses, and recent enrollment activity.

### Open the Demo Readiness Page

1. From the sidebar or dashboard, open **Demo Readiness**.
2. Review completed modules.
3. Review sample accounts.
4. Review institutional seed coverage.
5. Review live demo risk checks.
6. Follow the recommended demo flow.

### Manage Students

1. Open **Students** or `/admin/students`.
2. Search by name, email, student ID, or registration number.
3. Filter students by faculty, department, program, or status.
4. Select a student to view profile details.
5. Review student ID, registration number, faculty, department, program, level, guardian details, and verification status.
6. Activate or suspend student accounts if needed.
7. Open a student academic records page where applicable.

### Manage Courses

1. Open **Manage Courses** or `/admin/courses`.
2. Search by course title or code.
3. Filter by faculty, department, program, and active status.
4. View course details, credit hours, semester, level, and enrollment count.
5. Edit course information if needed.
6. Deactivate or reactivate courses.

### View Course Details

1. Open a course from the course management list.
2. Review the course summary.
3. Review enrollment counts by status.
4. Review recent enrollments for that course.

### Review Enrollment Summaries

1. Open **Enrollments** or `/admin/enrollments`.
2. Review the queue summary.
3. Filter by pending, approved, rejected, correction required, or all statuses.
4. Open enrollment details to review student profile, selected courses, credits, and decision history.

### Check System Readiness Before Demo

1. Open `/admin/demo-readiness`.
2. Confirm seed coverage is complete.
3. Confirm sample accounts are available.
4. Confirm demo steps are understood.
5. Confirm backend health check and frontend deployment are working.

## 5. Registrar Guide

### Login as Registrar

1. Open the UniPortal frontend URL.
2. Login using the demo registrar email.
3. Use the registrar password printed by the seed script.

### Open Registrar Dashboard

1. Open `/registrar/dashboard`.
2. Review pending enrollments.
3. Review approved, rejected, and returned enrollment totals.
4. Review approvals, rejections, and returns for the current day.
5. Confirm the active academic session and enrollment status.

### View Pending Enrollments

1. Open **Enrollments** or `/admin/enrollments`.
2. Ensure the status filter is set to pending.
3. Review student information, academic profile, selected courses, and credit totals.
4. Open **Detail Review** for a full enrollment view.

### Approve Enrollments

1. Open an enrollment detail page.
2. Confirm student information is complete.
3. Confirm selected courses are valid.
4. Confirm credit load is acceptable.
5. Click approve.
6. Confirm the enrollment status changes to approved.

### Reject Enrollments

1. Open an enrollment detail page.
2. Select a structured rejection reason.
3. Add custom text if needed.
4. Submit the rejection.
5. Confirm the student can see the rejection status and reason.

### Return Enrollments for Correction

1. Open an enrollment detail page.
2. Select a correction reason.
3. Add action-needed text if useful.
4. Return the enrollment for correction.
5. Confirm the status changes to correction required.

### View Decision Audit Log

1. Open an enrollment detail page.
2. Scroll to the registrar decision history or audit log section.
3. Confirm the log shows:
   - reviewer
   - decision date
   - previous status
   - new status
   - reason

## 6. Student Guide

### Login as Student

1. Open the UniPortal frontend URL.
2. Login with a demo student email.
3. Use the password printed by the seed script.

### Update Profile

1. Open **Profile** or `/profile`.
2. Review personal and academic information.
3. Update allowed profile fields.
4. Save changes.

Some academic fields may be protected after admin verification.

### Browse Courses

1. Open **Courses** or `/courses`.
2. Review the active academic session.
3. Review available courses.
4. Confirm whether registration is open or closed.

### Submit Enrollment

1. On the Courses page, select a course.
2. Click request enrollment.
3. Confirm the request is submitted.
4. Repeat for other allowed courses if needed.

Enrollment may be blocked if:

- registration is closed
- the course is inactive
- the course belongs to the wrong semester
- the student exceeds the credit limit
- the student already submitted the same course

### View Enrollment Status

1. Open **Enrollment Slips** or `/my-enrollments`.
2. Review academic year, semester, status, course count, and credits.
3. Open an enrollment to view details.

### Print Approved Enrollment Slip

1. Open an approved enrollment from `/my-enrollments`.
2. Review student details, academic session, approved courses, total credits, approval status, decision date, and reviewer.
3. Click print slip.
4. Confirm the print layout is clean.

Printable slips are only available for approved enrollments.

### View Academic Records

1. Open **Academic Records** or `/my-academic-records`.
2. Review academic year and semester.
3. Review courses, credit hours, grades, grade points, GPA, and remarks.

## 7. Recommended Live Demo Flow

Use this flow for a clear institutional presentation:

1. Start with admin login.
2. Show the admin dashboard and institutional snapshot.
3. Open Demo Readiness and explain the completed modules.
4. Show Student Management and student academic identity fields.
5. Show Course Management and filters by faculty, department, and program.
6. Open a course detail page and show enrollment counts.
7. Switch to a student account.
8. Browse courses and submit or view an enrollment request.
9. Open My Enrollments and explain status tracking.
10. Switch to registrar.
11. Open Registrar Dashboard.
12. Open Enrollment Review.
13. Approve, reject, or return an enrollment for correction.
14. Show the decision audit log.
15. Return to the student account.
16. Show approved enrollment slip printing.
17. Show student academic records.

## 8. Troubleshooting

### Cannot Login

- Confirm demo seed was run.
- Confirm you are using the correct demo email.
- Confirm you copied the password from the seed terminal output.
- Confirm the frontend is connected to the correct backend.

### API URL Not Connected

- Confirm `VITE_API_URL` is set in Vercel.
- It should include `/api/v1`.
- Example format: `https://your-render-service.onrender.com/api/v1`.

### Render Service Sleeping

- Free Render services may sleep when inactive.
- Open the backend health check URL.
- Wait for the service to wake up.
- Retry login after the backend responds.

### CORS Error

- Confirm backend `CLIENT_URL` matches the exact Vercel frontend origin.
- Do not include extra paths.
- Remove trailing slashes if needed.
- Use `CORS_ORIGINS` only when supporting multiple frontend origins.

### Missing Demo Data

- Run the seed script again:
  ```bash
  cd server
  npm run seed:demo
  ```
- Confirm the seed is running against the same database used by the deployed backend.

### 401 Unauthorized

- The user is not logged in or the token expired.
- Logout and login again.
- Confirm the browser has the latest token.

### 403 Forbidden

- The logged-in user does not have permission for that page.
- Admin-only pages require an admin account.
- Registrar pages require registrar or admin access.
- Student pages require a student account.

### Enrollment Not Showing

- Confirm the student has submitted an enrollment.
- Confirm you are viewing the correct academic year and semester.
- Confirm the registrar queue filter is not hiding the enrollment.

### Wrong Production Database

- Confirm Render `DATABASE` points to the intended MongoDB Atlas database.
- Confirm seed data was loaded into that same database.
- Confirm Vercel `VITE_API_URL` points to the same Render backend.

## 11. Demo Positioning Note

For a live university presentation, describe UniPortal as:

> A focused institutional MVP for academic setup, student management, course registration, registrar enrollment approval, enrollment slips, and academic record foundations.

Avoid presenting the current demo as a complete production SIS. Admissions, fees, timetable, lecturer result entry, full transcript processing, exports, communication, and attendance are recommended next-phase modules.
